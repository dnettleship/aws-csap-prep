Networking

AWS Direct Connect

Dedicated private connection from on-premises to AWS — consistent bandwidth, lower latency

Connection Types

TypeSpeedSetup TimeNotes
Dedicated1, 10, 100 GbpsWeeksPhysical cross-connect; capacity is yours alone
Hosted (via partner)50 Mbps – 10 GbpsDaysShares physical capacity with other customers

Virtual Interfaces (VIFs)

  • Private VIF — connects to a single VPC via a Virtual Private Gateway (VGW). One VIF → one VPC.
  • Public VIF — access AWS public service endpoints (S3, DynamoDB, API endpoints) over DX, not the internet
  • Transit VIF — connects to a Direct Connect Gateway, which then connects to Transit Gateways across multiple regions. Required for multi-VPC/multi-region topology.

Direct Connect Gateway

  • Global resource (not region-specific)
  • One DX connection → DX Gateway → VPCs in multiple AWS regions
  • Cannot route between VPCs through the DX Gateway — only on-prem ↔ VPC traffic
  • Can be associated with multiple VGWs or TGWs

Resilience Patterns

LevelConfiguration
Basic1 DX connection (single point of failure)
High (recommended)2 DX connections at 2 different DX locations
Maximum2 connections per location × 2 locations = 4 connections
  • Always configure a Site-to-Site VPN as DX fallback
  • BGP: set lower local-preference on VPN paths so DX is preferred
  • LAG (Link Aggregation Group): bundle multiple DX connections at the same location for higher throughput and redundancy

Encryption

DX is not encrypted by default — it is a private physical wire, but not cryptographically protected. Two options:

  1. VPN over Public VIF — IPSec tunnel over the DX public VIF. Encrypted + uses AWS backbone.
  2. MACsec (802.1AE) — Layer 2 hardware encryption. Only available on dedicated 10/100 Gbps connections.

Monitoring

  • ConnectionState — up/down
  • ConnectionBpsEgress / ConnectionBpsIngress — throughput metrics
  • For multi-region access via a single DX: Transit VIF → DX Gateway → TGW (per region)
  • DX does not encrypt — the most commonly tested trap; add VPN or MACsec when compliance requires encryption in transit
  • Private VIF → DX Gateway can reach multiple VPCs but only within the same region as the VGW; for multi-region you need Transit VIF → TGW

CloudFront & Global Accelerator

CDN caching, edge functions, origin failover, Anycast acceleration

CloudFront Fundamentals

  • Edge Locations (~450+) cache content close to users; Regional Edge Caches sit between edge locations and origins
  • Supported origins: S3, ALB, EC2, API Gateway, any HTTP server; multiple origins per distribution
  • Reduces origin load and egress costs — data transfer from AWS origins to CloudFront is free; only edge-to-user charges apply
  • Cache key: URL path + optional headers, cookies, query strings — configure with Cache Policies

Cache Control

MechanismDetails
TTLDefault 86400s; origin Cache-Control header overrides; min/max TTL set in Cache Policy
InvalidationPurge cached objects by path or wildcard (e.g. /images/*); first 1,000 paths/month free
Versioned URLsPreferred over invalidations — change filename/query string to force new cache key

Origin Failover

  • Configure an origin group with a primary and secondary origin
  • On 5xx or specified HTTP error codes from primary, CloudFront automatically retries against the secondary
  • Use for active-passive HA: S3 primary + S3 secondary in another region, or two ALBs

Security Features

  • HTTPS: CloudFront terminates TLS at the edge; can enforce HTTPS-only and HTTP→HTTPS redirect
  • AWS WAF: attach a Web ACL to a distribution for L7 filtering at the edge
  • AWS Shield Standard: included automatically; Shield Advanced adds DDoS response team + cost protection
  • Signed URLs / Signed Cookies: restrict access to authenticated users; signed URLs for single objects, signed cookies for multiple objects
  • Origin Access Control (OAC): replaces legacy OAI; allows CloudFront to access private S3 buckets without making them public
  • Field-Level Encryption: encrypt specific POST fields at the edge using public key; decrypted only at origin

Global Accelerator

  • Provides two static Anycast IPs; traffic enters the AWS network at the nearest edge PoP, then travels the private AWS backbone to the origin
  • Works at Layer 3/4 (TCP/UDP) — not a CDN; does not cache content
  • Ideal for non-cacheable dynamic content: APIs, gaming, VoIP
  • Supports health checks + automatic failover between endpoints across regions
  • Weighted traffic dials: shift traffic percentage between endpoints for canary or blue/green
  • Also supports client affinity: route the same client IP to the same endpoint consistently

Common Exam Scenarios

ScenarioAnswer
Serve private S3 content through CloudFront without making bucket publicOrigin Access Control (OAC) + bucket policy
Reduce global API latency, responses are not cacheableGlobal Accelerator
Block malicious IPs at the CDN edgeCloudFront + AWS WAF Web ACL
Automatic failover if primary S3 origin returns errorsCloudFront origin group with secondary origin
Run auth logic (JWT validation) before CloudFront forwards to originLambda@Edge on viewer request
App must have fixed IPs for firewall whitelisting + global low latencyGlobal Accelerator (provides static Anycast IPs)
Reduce S3 data transfer costs for global static asset deliveryCloudFront — origin-to-edge transfer is free
  • CloudFront caches at the edge — use invalidations sparingly; versioned file names are preferred
  • Global Accelerator does not cache — every request reaches the origin
  • OAC replaces OAI for new distributions; OAI is legacy but still on the exam
  • Lambda@Edge runs in all edge locations globally — it cannot access VPC resources or use environment variables in the same way as regular Lambda
  • CloudFront signed URLs expire — not suitable for long-lived public asset delivery; use only for time-limited private access

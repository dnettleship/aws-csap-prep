Lambda & Step Functions

Serverless execution, concurrency, cold starts, event sources, orchestration

Lambda Fundamentals

  • Max execution time: 15 minutes — not suitable for long-running jobs; use ECS/Fargate or Step Functions
  • Memory: 128 MB – 10,240 MB; CPU allocation scales linearly with memory — increase memory to get more CPU
  • Ephemeral storage (/tmp): up to 10,240 MB; lost after invocation
  • Deployment package: up to 50 MB zipped (250 MB unzipped); use Lambda Layers or container images (up to 10 GB) for larger dependencies
  • Runs in AWS-managed VPC by default; attach to your VPC for private resource access (adds cold start latency)

Concurrency

  • Default concurrent executions: 1,000 per region (across all functions); soft limit — can be increased via support
  • Reserved concurrency: guarantees N executions for a function; also caps it at N — prevents one function from consuming all regional capacity
  • Provisioned Concurrency: pre-initialised execution environments; eliminates cold starts; pay even when idle; use for latency-sensitive functions
  • When concurrency limit is hit: synchronous invocations return 429 (throttled); SQS/Kinesis triggers retry; async invocations go to DLQ

Cold Starts

  • First invocation (or after idle period) requires initialising a new execution environment — adds 100ms–1s+ latency
  • Minimise with: smaller packages, compiled runtimes (Rust > Go > Java > Python), Provisioned Concurrency
  • VPC-attached Lambda cold starts are higher (ENI creation) — mitigated since 2019 with shared ENI model, but still higher than non-VPC

Invocation Models

ModelBehaviourExamples
SynchronousCaller waits for response; errors returned to callerAPI Gateway, ALB, SDK direct invoke
AsynchronousLambda queues the event; retries twice on failure; DLQ or destination on failureS3 events, SNS, EventBridge, CloudWatch Events
Event source mappingLambda polls the source; processes batches; source controls retry behaviourSQS, Kinesis, DynamoDB Streams, MSK, MQ

Event Source Mapping — Key Settings

  • Batch size: number of records sent per Lambda invocation; larger batches = fewer invocations = more efficient
  • Bisect on error: split a failed batch in half to isolate poison pill records (Kinesis / DynamoDB Streams)
  • Maximum retry attempts / bisect: configure failure handling before moving records to DLQ/destination
  • Filter criteria: process only matching events (reduces unnecessary invocations)

Destinations & DLQ

  • Async destinations: on success or failure, send invocation record to SQS, SNS, EventBridge, or another Lambda
  • DLQ: legacy mechanism for failed async invocations; SQS or SNS; captures payload only
  • Prefer Destinations over DLQ — richer context, success + failure routing

Layers & Container Images

  • Layers: shared code/dependencies as a zip; up to 5 layers per function; total unzipped ≤ 250 MB
  • Container images: package function as Docker image up to 10 GB; use for large ML models or complex dependencies

Step Functions

  • Serverless orchestration for multi-step workflows; state machine defined in Amazon States Language (JSON/YAML)
  • Standard workflows: durable; up to 1 year; exactly-once execution; audit history in console; pay per state transition
  • Express workflows: high-throughput; up to 5 minutes; at-least-once; pay per execution duration; CloudWatch Logs for audit
  • Integrates with Lambda, ECS, DynamoDB, SQS, SNS, Bedrock, HTTP endpoints, and more — without writing glue code
  • Use for: data pipelines, order processing, ML training pipelines, human approval workflows
  • Wait for task token: pause workflow until an external system sends a callback — use for human approvals or async integrations

Large ML model inference (5 GB model file)Lambda container image (up to 10 GB)
  • Lambda max timeout is 15 minutes — any workload exceeding this needs ECS, Fargate, or Batch
  • Increasing memory also increases CPU — if a function is CPU-bound and slow, double the memory first
  • Provisioned Concurrency is billed even when idle — cost trade-off vs cold start latency
  • Lambda in a VPC requires a NAT Gateway or VPC endpoints to reach AWS services outside the VPC
  • Step Functions Express workflows are at-least-once — not suitable for non-idempotent operations

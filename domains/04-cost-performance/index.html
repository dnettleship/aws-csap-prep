7 Rs, migration services, database migration patterns, modernisation, hybrid connectivity

AWS Migration Services

  • AWS MGN (Application Migration Service) — agent-based continuous block-level replication; near-zero cutover downtime; replaces deprecated SMS
  • AWS DMS (Database Migration Service) — homogeneous and heterogeneous DB migration; supports ongoing CDC replication
  • AWS SCT (Schema Conversion Tool) — converts schema and stored procedures for heterogeneous migrations (e.g., Oracle → Aurora PostgreSQL)
  • AWS DataSync — online file data transfer: NFS/SMB/HDFS → S3/EFS/FSx; scheduled or one-time
  • AWS Snow family — offline transfer: Snowcone (8 TB), Snowball Edge (80 TB), Snowmobile (100 PB)

Database Migration Patterns

  • Same engine (MySQL → RDS MySQL): DMS full load + CDC replication until cutover
  • Different engine (Oracle → Aurora PostgreSQL): SCT first (schema + stored procedures), then DMS (data + CDC)
  • Large databases: Snowball Edge for initial bulk load, then DMS CDC for delta replication
  • Zero-downtime: keep DMS ongoing replication running until cutover window
Key trap: DMS does not migrate stored procedures, triggers, or views automatically — use SCT for those. DataSync is for file data only, not databases.

Modernisation Patterns

  • Strangler Fig — incrementally replace the monolith; use ALB path-based routing to divert specific endpoints to new microservices, leaving the rest to the monolith
  • Microservices — decompose by bounded context; ECS/EKS + SQS/EventBridge for async communication
  • Containerisation — AWS App2Container (A2C) for Java/.NET apps; outputs ECS/EKS task definitions automatically
  • Serverless — move to Lambda for event-driven workloads; eliminate server management

Hybrid Connectivity

  • Direct Connect — dedicated 1/10/100 Gbps; consistent bandwidth; not encrypted by default
  • Direct Connect Gateway — single DX connection to multiple VPCs across multiple regions via Transit VIF
  • Site-to-Site VPN — IPSec over internet; set up in hours; use as DX backup
  • VPN over DX — adds IPSec encryption over the private DX connection (compliance requirement)

Migration Phases

  1. Assess — portfolio discovery, TCO analysis (Migration Evaluator), readiness assessment
  2. Mobilise — landing zone, migration factory setup, runbooks
  3. Migrate & Modernise — execute migrations in waves, iterate

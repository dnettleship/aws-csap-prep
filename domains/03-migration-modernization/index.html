Compute Optimisation

OptionDiscountBest For
Compute Savings Plansup to 66%Flexible: any instance family, region, OS, Fargate, Lambda
EC2 Instance Savings Plansup to 72%Committed to family + region; less flexible
Reserved Instancesup to 75%Stable, predictable workloads; Standard RIs can be sold on Marketplace
Spot Instancesup to 90%Fault-tolerant, stateless, interruptible workloads
  • Right-size with AWS Compute Optimizer (ML-based recommendations for EC2, Lambda, ECS, EBS)
  • Graviton (ARM) instances: 20–40% better price/performance for most workloads
  • Lambda: increase memory to increase CPU allocation; Provisioned Concurrency eliminates cold starts

Storage Cost Optimisation

  • S3 Intelligent-Tiering: auto-moves objects; per-object monitoring fee — not suitable for objects < 128 KB
  • S3 Lifecycle policies: automate transitions (Standard → IA → Glacier → Deep Archive)
  • EBS: gp3 is cheaper than gp2 at the same performance; delete unattached volumes; use Snapshot Lifecycle Manager

Networking Cost Optimisation

  • Same-AZ traffic is free; cross-AZ is charged; cross-region is charged
  • VPC Gateway Endpoints for S3 and DynamoDB are free — eliminate NAT Gateway processing charges for those services
  • NAT Gateway: charged per GB processed + per hour — most expensive for high-volume internal AWS traffic
  • CloudFront: reduces origin data transfer costs via edge caching

Database Performance & Cost

  • RDS Read Replicas: offload reads; up to 15 Aurora replicas; can promote for DR
  • ElastiCache: cache frequent reads; Redis for persistence/pub-sub; Memcached for simple cache
  • DynamoDB DAX: microsecond reads; fully API-compatible in-memory cache
  • Aurora Parallel Query: pushes analytical processing to storage layer
  • Aurora Serverless v2: 0.5 ACU minimum; dev/test or sporadic workloads
  • RDS: stop instances when not needed; right-size via Performance Insights

Observability & Monitoring

  • CloudWatch: metrics, alarms, dashboards, Logs Insights; use metric math for composite metrics
  • CloudWatch Container Insights: ECS/EKS performance monitoring
  • AWS X-Ray: distributed tracing across services; identify latency bottlenecks
  • AWS CloudTrail: API activity logs for security and audit
  • VPC Flow Logs: network traffic metadata for troubleshooting and security

Well-Architected Framework

Five pillars — each represents an axis of improvement for existing solutions:

  • Operational Excellence: IaC (CloudFormation/CDK), runbooks, post-incident reviews
  • Security: least-privilege IAM, encryption at rest/in transit, GuardDuty, Security Hub
  • Reliability: Multi-AZ, Auto Scaling, backups, chaos engineering
  • Performance Efficiency: right-sizing, caching, serverless, placement groups
  • Cost Optimisation: Savings Plans, Spot, lifecycle policies, Cost Explorer

Cost Governance

  • Cost Explorer: visualise, filter, forecast; RI and Savings Plans recommendations
  • AWS Budgets: alerts + Budgets Actions (apply SCPs, stop EC2/RDS) when thresholds are hit
  • Cost Allocation Tags: activate in Billing; group costs by team/project/environment
  • Trusted Advisor: cost optimisation checks — idle resources, underutilised EC2, unassociated EIPs

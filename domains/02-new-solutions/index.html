Domain 2 — 29%

Design for New Solutions

Highest-weighted domain — compute, storage, databases, decoupling, security, HA design

Compute Strategy

  • EC2: placement groups (cluster/spread/partition), instance types, launch templates
  • ECS vs EKS: ECS is AWS-native and simpler; EKS when Kubernetes portability or ecosystem is needed
  • Lambda: event-driven, max 15 min, 10 GB memory; ideal for async/decoupled patterns
  • Fargate: serverless containers; no node management; use to avoid EC2 operational overhead

Storage Design

  • S3: storage classes (Standard → IA → Glacier Instant/Flexible/Deep Archive), lifecycle policies, Object Lock (WORM), Replication (SRR/CRR)
  • EFS: shared POSIX filesystem; multi-AZ; use for shared state across EC2/ECS/Lambda
  • FSx: FSx for Windows (SMB/AD), FSx for Lustre (HPC/ML), FSx for NetApp ONTAP
  • EBS: gp3 default; io2 Block Express for sub-ms latency; snapshots cross-region

Decoupling & Async Patterns

  • SQS: pull-based, up to 14-day retention; Standard (at-least-once) vs FIFO (exactly-once, 3,000 TPS with batching)
  • SNS: push/fan-out to SQS, Lambda, HTTP, email; filter policies reduce noise
  • EventBridge: event bus with rules; use for SaaS integrations, cross-account events, and dynamic routing
  • Kinesis Data Streams: real-time, ordered per shard, replay up to 365 days
  • Kinesis Firehose: managed delivery to S3/Redshift/OpenSearch; no replay

Security Architecture

  • KMS: key policy AND IAM policy both required for access; customer-managed CMKs for fine-grained control
  • Secrets Manager: automatic rotation; use over Parameter Store when rotation is required
  • Parameter Store: SecureString (KMS-encrypted); cheaper for static config/secrets
  • ACM: managed TLS certificates; free for ALB/CloudFront; cannot export private key
  • WAF: L7 rules on CloudFront, ALB, API Gateway; managed rule groups for OWASP top 10

High Availability Design

  • Multi-AZ for stateful services (RDS Multi-AZ, EFS, ALB)
  • Multi-region for DR (pilot light, warm standby, active-active)
  • Auto Scaling: target tracking preferred for steady-state; scheduled for predictable peaks
  • ALB > NLB for HTTP/S; NLB for ultra-low latency or static IP; GLB for inline security appliances

Design Solutions for Organizational Complexity

Multi-account strategy, SCPs, Control Tower, cross-account networking and security

Multi-Account Strategy

  • AWS Organizations — management account, OUs, SCPs, consolidated billing
  • AWS Control Tower — landing zone, guardrails (preventive vs. detective), Account Factory
  • AWS IAM Identity Center (SSO) — centralised access across accounts via permission sets
  • Recommended account separation: security, log archive, network, shared services, workloads (dev/staging/prod)

Service Control Policies (SCPs)

  • SCPs are permission boundaries — they restrict what IAM can grant; they don't grant anything themselves
  • Applied at root, OU, or account level; inherited down the hierarchy
  • Management account is never restricted by SCPs
  • Service-linked roles are not restricted by SCPs

Common SCP Patterns

Deny actions outside approved regions

{
  "Effect": "Deny",
  "Action": "*",
  "Resource": "*",
  "Condition": {
    "StringNotEquals": {
      "aws:RequestedRegion": ["eu-west-1", "eu-west-2"]
    }
  }
}

Prevent disabling CloudTrail

{
  "Effect": "Deny",
  "Action": ["cloudtrail:StopLogging", "cloudtrail:DeleteTrail"],
  "Resource": "*"
}

Cross-Account Access Patterns

  • IAM role assumption (sts:AssumeRole) — preferred over long-lived IAM users in each account
  • Resource-based policies (S3, KMS, SNS, SQS, Lambda) — allow cross-account without role switch
  • AWS RAM — share VPCs, subnets, Transit Gateways, Route 53 Resolver rules across accounts

Centralised Logging & Security

  • CloudTrail — organisation trail logs all member accounts to central S3 bucket in log archive account
  • AWS Config — aggregator collects compliance data from all accounts and regions
  • Security Hub — aggregated findings via delegated admin; integrates GuardDuty, Inspector, Macie
  • GuardDuty — organisation-level enablement; delegate admin to security tooling account
Gotcha: GuardDuty findings in member accounts are visible in the delegated admin account, not the management account. These are different things.

Networking at Scale

  • Transit Gateway — hub-and-spoke; replaces VPC peering mesh; transitive routing; share via RAM
  • AWS Network Firewall — centralised deep packet inspection in a shared networking account
  • VPC sharing via RAM — centralised VPC, spoke accounts deploy resources into shared subnets; owner controls routing

Cheatsheet

Common Exam Traps

30+ gotchas that the SAP-C02 uses as distractors — read these before your exam

IAM & Security

  • SCPs don't apply to the management account — even a Deny at root doesn't affect it
  • SCPs don't grant permissions — they only restrict; you still need IAM policies to allow actions
  • Resource-based policies can grant cross-account access without role assumption — an S3 bucket policy granting another account access means no role switch is needed
  • KMS: key policy AND IAM policy must both allow — even if IAM permits kms:Decrypt, the key policy must also allow it
  • Secrets Manager vs Parameter Store — Secrets Manager for automatic rotation; Parameter Store for static config/secrets
  • Cognito User Pools vs Identity Pools — User Pools = authentication (who you are); Identity Pools = exchange OIDC/SAML token for AWS credentials

Networking

  • VPC peering is non-transitive — A↔B and B↔C does not enable A↔C; use Transit Gateway for transitive routing
  • Direct Connect is not encrypted — it's a private wire, not cryptographically protected; add VPN or MACsec for encryption
  • Security Groups are stateful; NACLs are stateless — SG return traffic is automatically allowed; NACLs need explicit rules for both directions
  • NACLs evaluate rules in numerical order; first match wins — lower rule number takes precedence
  • NAT Gateway is AZ-specific — create one per AZ for HA; each AZ's private subnets should route to their own NAT GW
  • VPC Gateway Endpoints for S3 and DynamoDB are free — interface endpoints cost money; gateway endpoints don't

Compute

  • Spot interruption gives a 2-minute warning — not immediate; use this window to checkpoint state or drain connections
  • Lambda maximum execution time is 15 minutes — not suitable for long-running jobs; use ECS/Fargate or Step Functions
  • Lambda concurrent executions default limit is 1,000 per region — soft limit; can be increased; applies across all functions in the region
  • EC2 Auto Scaling default cooldown is 300 seconds — prevents rapid scale-in/out; Target Tracking manages its own scale-in cooldown

Storage & Databases

  • RDS Multi-AZ standby is NOT readable — it exists for failover only; use read replicas for read scaling
  • Aurora read replicas ARE automatic failover targets — they serve reads AND replace the primary on failure (unlike RDS read replicas)
  • S3 is strongly consistent since December 2020 — the old "eventual consistency for overwrites" trap is outdated
  • DynamoDB hot partition — design partition keys for uniform distribution; sequential IDs (e.g., timestamps) concentrate load on one partition
  • ElastiCache Redis vs Memcached — Redis: persistence, replication, pub/sub, sorted sets; Memcached: simple multi-threaded cache, no persistence

Migration

  • AWS MGN replaces SMS — Server Migration Service (SMS) is deprecated; use MGN for all lift-and-shift migrations
  • DMS doesn't migrate stored procedures, triggers, or views — use SCT for schema and code conversion
  • DataSync is for file and object data, not databases — use DMS for database migrations
  • Snowball data lands in S3 Standard — add a lifecycle policy if Glacier storage pricing is needed

Cost

  • Reserved Instances cannot be cancelled — standard RIs can be sold on the RI Marketplace; convertible RIs cannot
  • Compute Savings Plans are more flexible than EC2 Instance Savings Plans — but have a lower maximum discount ceiling
  • S3 Intelligent-Tiering has a per-object monitoring fee — not cost-effective for objects under 128 KB
  • Data transfer into AWS is always free — only egress to the internet is charged

High Availability

  • "Highly available" on the exam means survive an AZ failure — minimum two AZs; multi-region is a DR concern
  • Multi-AZ ≠ Multi-Region — Multi-AZ protects against AZ failure; multi-region protects against regional failure
  • ALB is inherently multi-AZ — you don't configure HA for ALB; AWS manages it

Common Distractors

  • "Most cost-effective" + "highly available" → Reserved/Savings Plans + Multi-AZ managed services
  • "Regulatory/compliance" + "encryption in transit" over Direct Connect → VPN over DX (IPSec), not just DX alone
  • "Existing VMware" on-premises → VMware Cloud on AWS (Relocate strategy)
  • "Minimal downtime" server migration → AWS MGN (continuous replication, near-zero cutover window)
  • "Petabytes" of data to transfer → AWS Snowball Edge / Snowmobile, not DataSync over the network

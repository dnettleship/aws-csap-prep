S3 Storage Class Decision Tree

Pick the right storage class; know the minimum durations, retrieval times, and cost traps

Quick Reference

Storage ClassRetrievalMin DurationMin SizeUse Case
S3 StandardInstantNoneNoneFrequently accessed data
S3 Standard-IAInstant30 days128 KBInfrequent access; disaster recovery copies
S3 One Zone-IAInstant30 days128 KBInfrequent, re-creatable data; 20% cheaper than Standard-IA
S3 Intelligent-TieringInstant (frequent/infrequent tiers); minutes (archive tiers)NoneNone (monitoring fee applies)Unknown or changing access patterns
Glacier Instant RetrievalInstant (ms)90 days128 KBArchives accessed roughly once per quarter
Glacier Flexible RetrievalMinutes – 12 hours90 days40 KBArchives; bulk expedited options; no instant need
Glacier Deep Archive12 – 48 hours180 days40 KB7–10 year compliance archives; lowest cost storage

Decision Tree

How often is this data accessed?

  Frequently (daily/weekly)
    → S3 Standard

  Infrequently (monthly)
    → Need instant retrieval?
        Yes → S3 Standard-IA  (or One Zone-IA if data is re-creatable)
        No  → Glacier Instant Retrieval (quarterly access pattern)

  Rarely / Archive
    → Can you wait minutes to hours?
        Yes → Glacier Flexible Retrieval
        No  → Glacier Instant Retrieval

  Long-term compliance (7+ years), hours of retrieval acceptable
    → Glacier Deep Archive

  Access pattern unknown or varies over time
    → S3 Intelligent-Tiering

Lifecycle Policy Example

Day 0    → S3 Standard          (active data)
Day 30   → S3 Standard-IA       (cooling off)
Day 90   → Glacier Flexible     (archive)
Day 365  → Glacier Deep Archive (long-term)
Day 2555 → Expire / delete      (7-year retention met)

Key Rules

  • Minimum storage duration is charged even if you delete the object earlier
  • Standard-IA and One Zone-IA have a per-GB retrieval fee on top of storage — not suitable for frequent access
  • Intelligent-Tiering: per-object monitoring fee (~$0.0025 / 1,000 objects/month); objects < 128 KB are never moved out of frequent tier — makes it uneconomical for small objects
  • Cannot transition FROM Glacier back to Standard via lifecycle — restore to a temporary Standard copy first
  • One Zone-IA: single AZ — 20% cheaper but risk of permanent data loss if that AZ fails

S3 Object Lock (WORM)

  • Governance mode: objects protected; users with s3:BypassGovernanceRetention permission can override
  • Compliance mode: no one, including root, can delete before retention period expires
  • Use for: SEC Rule 17a-4, HIPAA, CJIS, financial services compliance
  • Snowball / Snow device data lands in S3 Standard only — add a lifecycle policy if Glacier pricing is needed
  • S3 Intelligent-Tiering monitoring fee makes it uneconomical for objects < 128 KB — they never move out of frequent tier anyway

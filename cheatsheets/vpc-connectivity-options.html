Key Routing Rules

  • Route tables are subnet-level, not VPC-level
  • More-specific routes win (longest prefix match)
  • VPC peering routes must be added manually to route tables in both VPCs
  • TGW route propagation is automatic when enabled on an attachment
  • NAT Gateway: one per AZ for HA — route each AZ's private subnet to its own NAT GW
